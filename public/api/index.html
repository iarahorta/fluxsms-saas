Documentação da API

Referência das rotas HTTP expostas pelo backend FluxSMS para o ecossistema de parceiros (chave de API no worker desktop) e para o portal web autenticado com sessão Supabase (JWT). Estrutura inspirada em páginas de referência rápida como a documentação pública de provedores de SMS.

https://fluxsms.com.br

Em staging pode usar o host Railway configurado no CORS. Todas as rotas usam HTTPS em produção.

Autenticação

Partner API (/partner-api/* exceto /docs): envie a chave em um dos formatos:

  • Cabeçalho x-api-key: <sua_chave>
  • Ou Authorization: Bearer <sua_chave>
  • Opcional: X-Flux-Hwid — na primeira chamada com HWID válido a chave pode ficar vinculada à máquina; depois é obrigatório.

Portal parceiro (/api/partner/*): Authorization: Bearer <access_token Supabase> do utilizador com perfil de parceiro.

Admin (/api/admin/*): JWT de utilizador com is_admin — não documentado aqui em detalhe.

Partner API — visão geral

Prefixo: /partner-api. A rota GET /partner-api/docs devolve JSON com a lista de endpoints (máquinas e integrações).

GET /partner-api/docs público

Índice JSON da API (sem autenticação).

GET /partner-api/health API key

Confirma chave válida, IP permitido e devolve partner_code e timestamp.

GET /partner-api/me API key

Perfil do parceiro (id, partner_code, margin_percent, etc.) e custos de serviço (costs).

GET /partner-api/worker/bootstrap API key

Resolve a estação (polo) preferencial e devolve polo.chave_acesso para configurar o worker local (POLO_KEY).

GET /partner-api/worker/summary API key

Resumo financeiro: commission_percent, saldo_total (disponível para solicitar saque), finance_totals.

Partner API — chips e fila

POST /partner-api/chips API key

Regista ou atualiza um chip no polo do parceiro. Corpo (aliases aceites):

{
  "porta": "COM3",
  "numero": "+5511999990000",
  "operadora": "Vivo"
}

Alternativas: port, number, operator. Respostas: 201 com chip, ou erros como em_quarentena (423).

POST /partner-api/worker/sync API key

Mesmo contrato que POST /partner-api/chips — canal usado pelo desktop após sync com o core.

GET /partner-api/worker/activations API key

Lista ativações em waiting para os chips do parceiro, enriquecidas com chip_porta / chip_numero. Respeita serviços desativados no painel.

GET /partner-api/worker/chips API key

Lista chips (id, porta, numero, status, …) e polos associados.

GET /partner-api/worker/chip-activations?porta=COM3 API key

Histórico de ativações do chip identificado pela porta (query obrigatória porta).

POST /partner-api/worker/heartbeat API key

Mantém polos do parceiro como ONLINE e atualiza last_ping em chips.

curl -s -X POST "https://fluxsms.com.br/partner-api/worker/heartbeat" \
  -H "x-api-key: SUA_CHAVE"
POST /partner-api/worker/shutdown API key

Fecho controlado do worker: polo e chips passam a estado offline no servidor.

Portal parceiro (JWT)

Prefixo: /api/partner/self. Sessão do painel em parceiros.fluxsms.com.br (ou portal forçado por env).

GET/api/partner/self/bootstrapJWT

Dados iniciais: utilizador, parceiro, chaves API ativas (plaintext só para chaves desencriptáveis), resumo financeiro, URL de download do worker.

GET/api/partner/self/chipsJWT

Chips e polos do parceiro com metadados de polo.

GET/api/partner/self/service-togglesJWT

Serviços (WhatsApp, Telegram, etc.) e flags enabled.

PUT/api/partner/self/service-togglesJWT

Corpo: { "services": [ { "service": "whatsapp", "enabled": false } ] }

POST/api/partner/self/api-keysJWT

Cria nova chave (opcional label). Limite de 3 chaves ativas por parceiro. Resposta inclui api_key em texto claro uma única vez.

Finanças parceiro (JWT)

Prefixo: /api/partner/finance.

GET/api/partner/finance/summaryJWT

Regras públicas (mínimo saque, taxa, carência, repasse %) e totais (repasse_liberado, disponivel_para_solicitar, …).

POST/api/partner/finance/withdrawJWT

Pedido de saque em horário comercial. Corpo exemplo:

{ "amount": 400, "pix_destination": "chave@email.com" }

Taxa fixa e janela horária podem devolver outside_business_hours ou validação de valor.

Health público

GET/healthpúblico

{ "status", "version", "ts" } — monitorização e load balancers.

SMS deliver (infraestrutura)

POST/sms/deliverHARDWARE_API_KEY

Uso interno do pipeline de modem / software autorizado — não é a Partner API Key. Autenticação com o mesmo padrão de cabeçalho, valor igual a HARDWARE_API_KEY no ambiente do servidor.

{ "activation_id": "uuid", "sms_code": "123456", "chip_porta": "COM3" }

JSON programático
